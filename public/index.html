2. How a website works?

Three Things of Website:

  • HTML: defines document structure with HTML elements.
  • CSS: defines views like colors, layout, width, etc.
  • JavaScript: defines actions like click events.

Tips

  • Use "View Page Source" to check css/javascript links in HTML
  • Use Browsers' Developer Tool
  • Check third-party developer tools (e.g. Firebug )

HTML Terms:

  • Element: a block of codes from start tag to end tag
  • Attribute: provides additional information/characteristics about an element
  • id: a unique identifier of an element
  • class: a shared identifier for grouping elements

DOM (Document Object Model)

  • When a browser loads a page, it converts HTML elements to DOM Node tree.
  • DOM Nodes are HTML Element objects.
  • HTML Elements objects, which we can use when lively manipulating HTML elements on a browser.

CSS Syntax

Selectors - Basic

  • Selectors: rules or patterns to select HTML Element objects
  • Select all elements by element tag name 
     e.g. div
  • Select the elements by id: 
     e.g. #sciug
  • Select all elements by class: 
     e.g. .sciug
Please Change Me! (Select ID and use a Developer Tool).

Selectors - Relationship

  • HTML elements are nested (e.g. html > body > article > section).
  • Select all descenents: 
     e.g. section p
  • Select direct childs only: 
     e.g. section > p
  • Select the first childs only: 
     e.g. h1:first-child
  • Select the nth child only: 
     e.g. p:nth-child(3)

Three Ways of Plugging CSS to HTML

  1. External: link tag in the the head element
  2. Internal: style tag in the head element
  3. Inline: style attribute in a HTML element (e.g. body, p, section, etc)
  4. External stylesheet is most preferred and popular.

CSS Priority/Specificity Rules (VERY IMPORTANT!)

  • Important when handling multiples stylesheets.
  • Rule 1: External >> Internal >> Inline
  • Rule 2: The later loaded one overrides the earlier loaded.
  • Rule 3: The specific rules override the general ones.
    • tag elements >> .class >> #id
    • closer relationship in nested HTML elements
  • Use a Developer Tool to find overridden styles

Why jQuery?

  • Library that helps to use JavaScript efficiently.
  • Selecting and manipulating elements, event handling, animation, Ajax.
  • Short code.
  • Overcome browser compatibility: Cross-Browser.
  • Extensible: plugins.

Ajax, Asynchronous Javascript and XML

  • A web development technique, not a lanuguage or a library
  • Interchange data with a server in JSON, XML, HTML, text.
  • Asychronous: Request a data update without reloading the entire page;
    this means that users can do other tasks waiting for the update.
  • Historically, developed by Miscrosoft to update web contents dynamically.
    In the early 2000's, adopted by Mozilla, Apple, and Google (e.g. Outlook Web, Gmail)
  • Core technology behind is XMLHttpRequest
  • Later in 2006, XMLHttpRequest standardized by W3C (World Wide Wieb Consoirtium)

Find:

  • Webforms are editable (manimenu, opacmenu, briefcit, bib_record, srchelp_X).
  • Some sections of search result page is not customizable.
  • Tokens are inserted into webforms.
  • Both toplogo and botlogo tokens are webforms.

Tokens:

  • A token contains a block of HTML code, mostly with data.
  • The html code inside tokens are not customizable: programtically generated.
  • A few flexible exceptions:
    • Webform token: toplogo, toplogo_loggedin, botlogo, etc
    • Web Options
    • Record data token
  • Some tokens allow record fields customizable.
    • e.g. recordinfo:ape, fieldspec:Vbt, linkfieldspec:Vbp
    • Important: Need to define record fields first in webpub.def file.
    • webpub.def: Record Display Definition file to define tags, order, and labels.
  • Token must take an entire line and left-aligned in a web form.

Web Options:

  • Controllers that define the looks and functions of WebPac.
  • Accessible from Web Options function in SDA.
  • Read Seirra Dcoumentation > Web Options
    • Lots of web options: not easy to find a right option.
    • Technical descriptions and definitions: not intuitive to undertand.
    • You may still need a call(s) to the Innovative.
  • Not always usable because of scope issue.
    • An option itself cannot be scoped.
    • Libraries' local situation: an option is shared.
  • My opinion: the options that control the views may be better excluded
    and they may be customizable inside web forms or stylesheet.

Editing Process

  • Make a plan from user experiences.
  • Find where your edit target is located.
  • Find a relevant webform and associated tokens.
  • Check whether it is customizable in web forms and record data tokens.
  • Check whether it is customizable in Web Options.
    (Read Sierra Documentation or call the Innovative.)
  • Check if it is customizable only for a specific scope.
  • Write CSS and JavaScript to manipuate HTML DOM elements.
  • Use WebPac staging host: Port 2082.
